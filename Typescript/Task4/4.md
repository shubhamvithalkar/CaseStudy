# let & const in TypeScript

## Problem Statement

Modern applications require safe and predictable handling of variables.

Some values should change during execution (like scores or counters).

Other values should remain constant (like configuration values or mathematical constants).

TypeScript provides `let` and `const` to manage variable behavior safely.

---

# Learning Objectives

By the end of this lesson you will:

- Declare variables using `let` and `const`
- Understand block scope
- Know when to use `let` vs `const`
- Avoid redeclaration and reassignment errors

---

# Declaring Variables with let

Syntax:

```ts
let var_name: type = value
```

Example:

```ts
let car_name: string = "Brezza"
let car_price: number = 1000000

console.log(car_name)
console.log(car_price)
```

Output

```
Brezza
1000000
```

---

# Block Scope

Variables declared with `let` exist only inside their block.

```ts
let bool: boolean = true

if (bool) {
    let result: number = 10
    console.log(result)
}

// console.log(result) // Error
```

---

# Redeclaration Rules

`let` variables cannot be redeclared in the same scope.

```ts
let animal: string = "cat"

// let animal: string = "dog" // Error
```

However, reassignment is allowed.

```ts
animal = "dog"
```

---

# Same Name in Different Blocks

```ts
let bool: boolean = false

if (bool) {
    let num: number = 1
    console.log(num)
} else {
    let num: number = 2
    console.log(num)
}
```

These variables are separate because they exist in different blocks.

---

# Declaring Variables with const

`const` declares a constant value that cannot be changed.

Syntax:

```ts
const var_name: type = value
```

Example:

```ts
const lang: string = "TypeScript"
const PI: number = 3.14

console.log(lang)
console.log(PI)
```

Output

```
TypeScript
3.14
```

---

# const Rules

- Must be initialized when declared
- Cannot be reassigned
- Cannot be redeclared
- Has block scope like `let`

Example:

```ts
if (true) {

    const PI: number = 3.14
    console.log(PI)

    // const PI = 3.14 // Error
    // PI = 3.15 // Error
}
```

---

# Best Practices

- Prefer `const` by default
- Use `let` only when value must change
- Avoid `var`
- Keep variables within the smallest possible scope

---

# Quick Comparison

| Feature | let | const |
|------|------|------|
| Value can change | Yes | No |
| Block scoped | Yes | Yes |
| Redeclaration allowed | No | No |
| Must initialize | No | Yes |
