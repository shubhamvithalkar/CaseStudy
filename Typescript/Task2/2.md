# Basic Syntax in TypeScript

## Problem Statement

When working in large software teams, code must be consistent, readable, and maintainable.

Developers need to follow clear syntax rules to avoid confusion and errors.

The challenge is learning the **basic syntax and structure of TypeScript** so code remains professional and easy to understand.

---

# Learning Objectives

By the end of this lesson you will:

- Understand TypeScript syntax basics
- Write variables, functions, and classes
- Use identifiers and keywords correctly
- Write comments for code explanation
- Compile and run TypeScript programs

Save file:

```

Compile:

Run:

```

---

# Identifiers Rules

- Can contain letters, numbers, `_`, `$`
- Cannot start with numbers
- Cannot contain spaces
- Cannot be reserved keywords
- Case sensitive

| Valid | Invalid |
|------|------|
| firstName | first name |
| first_name | 1number |
| num1 | first-name |
| $result | Var |

# Comments

### Single-line

```ts

```

### Multi-line

```ts

*/
```

# Object-Oriented Example

```ts
class Greeting {
  greet(): void {
    console.log("Hello World!!!");
  }
}

let obj = new Greeting();
obj.greet();
```

# Compiler Flags

| Flag | Description |
|----|----|
| --help | Show help |
| --module | Load modules |
| --target | Set JS version |
| --declaration | Generate type files |
| --removeComments | Remove comments |
| --out | Combine files |
| --sourcemap | Generate source maps |
| --noImplicitAny | Disallow untyped variables |
| --watch | Recompile on changes |

# Best Practices

- Use valid identifiers
- Add comments for complex logic
- Format and indent code
- Avoid reserved keywords
- Organize code using classes and functions
