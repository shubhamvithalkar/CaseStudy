# Built-in Types in TypeScript

## Problem Statement

Applications handle many types of data such as numbers, strings, booleans, and objects.

TypeScript provides built-in types to ensure data is stored and processed safely.

Using proper types helps prevent runtime errors and improves code reliability.

---

# Learning Objectives

By the end of this lesson you will:

- Understand TypeScript built-in types
- Declare variables with explicit types
- Handle special values such as null and undefined
- Write functions with proper return types

---

# Built-in Types Overview

| Type | Keyword | Description |
|---|---|---|
| Number | number | Numeric values |
| String | string | Text data |
| Boolean | boolean | True or false values |
| Void | void | Function returns nothing |
| Null | null | Intentional absence of value |
| Undefined | undefined | Variable not assigned |
| Symbol | symbol | Unique identifiers |
| Object | object | Structured data |
| Never | never | Function that never returns |

---

# Number

```ts
let age: number = 30
let marks: number = 30.5
let hex: number = 0xff
let binary: number = 0b1010
```

---

# String

```ts
let firstName: string = "John"
let lastName: string = "Doe"

let fullName: string = `${firstName} ${lastName}`
```

---

# Boolean

```ts
let isReady: boolean = true
```

---

# Symbol

```ts
const UNIQUE_KEY = Symbol()

let obj = {
  [UNIQUE_KEY]: "SecretValue"
}
```

---

# Null & Undefined

```ts
let empty: null = null
let undef: undefined
```

---

# Object

```ts
let person: object = {
  name: "Bob"
}
```

---

# Void

```ts
function log(): void {
  console.log("log")
}
```

---

# Never

```ts
function fail(): never {
  throw new Error("This always fails")
}
```

---

# Best Practices

- Always declare types for variables
- Handle null and undefined carefully
- Use never only for functions that never return
- Use objects for structured data

---

# Key Takeaways

- TypeScript built-in types help enforce strong typing
- Each type has a specific purpose
- Using correct types reduces runtime errors
