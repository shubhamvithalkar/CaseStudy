# Variables in TypeScript

## Problem Statement

In software systems we need to store information like names, scores, and settings.

Variables help store this data in memory with a name.

TypeScript helps ensure variables are used correctly through strong typing.

---

# Learning Objectives

By the end of this lesson you will:

- Understand variable declaration
- Learn naming rules
- Use type annotations and inference
- Understand variable scope
- Learn type assertions

---

# Naming Rules

Variable names:

- Can contain letters, numbers, `_`, `$`
- Cannot start with a number
- Cannot contain spaces
- Cannot be reserved keywords
- Are case-sensitive

| Valid | Invalid |
|---|---|
| firstName | first name |
| first_name | 1number |
| num1 | first-name |
| $result | var |

---

# Declaring Variables

## Type and Value

```ts
var name: string = "Mary"
```

## Type Only

```ts
var city: string
city = "Mumbai"
```

## Value Only (Type Inference)

```ts
var name = "Mary"
```

## No Type or Value

```ts
var data
```

---

# Example Program

```ts
var name: string = "John"
var score1: number = 50
var score2: number = 42.5

var sum = score1 + score2

console.log("name:", name)
console.log("first score:", score1)
console.log("second score:", score2)
console.log("sum:", sum)
```

---

# Strong Typing

```ts
var num: number = "hello" // Error
```

TypeScript prevents assigning wrong types.

---

# Type Assertion

```ts
var str = "1"
var num: number = <number><any>str
```

Type assertions only affect compile-time checking.

---

# Type Inference

```ts
var num = 2
```

TypeScript automatically infers the type.

---

# Variable Scope

## Global Scope

```ts
var globalNum = 12
```

## Class Scope

```ts
class Numbers {
  num_val = 13
}
```

## Local Scope

```ts
function test() {
  var localVar = 10
}
```

---

# Best Practices

- Use meaningful variable names
- Avoid using `any`
- Keep scope small
- Use type annotations
- Avoid reserved keywords
