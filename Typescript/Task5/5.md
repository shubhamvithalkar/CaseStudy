# The `any` Type in TypeScript

## Problem Statement

In dynamic systems like surveys or user forms, data can come in many shapes.

Some answers may be text, numbers, arrays, or objects.

To temporarily handle unknown data types, TypeScript provides the `any` type.

However, using `any` removes type safety and should be used carefully.

---

# Learning Objectives

By the end of this lesson you will:

- Understand the purpose of the `any` type
- Declare variables that accept any value
- Recognize the risks of using `any`
- Plan to transition to safer types later

---

# What is the `any` Type?

The `any` type allows a variable to store **any kind of value**.

TypeScript will **not perform type checking** on that variable.

---

# Declaring an any Variable

```ts
let answer: any
```

Assign different values:

```ts
answer = 42
answer = "blue"
answer = [1,2,3]
answer = { name: "Alex" }
```

---

# Example

```ts
let surveyAnswer: any

surveyAnswer = "Yes"
surveyAnswer = 5
surveyAnswer = ["Option A", "Option B"]
```

---

# Array of any

```ts
let allAnswers: any[] = []

allAnswers.push("No")
allAnswers.push(10)
allAnswers.push({ comment: "N/A" })
```

---

# Processing Answers

```ts
for (let ans of allAnswers) {
    console.log("Received answer:", ans)
}
```

---

# Risks of Using any

- Removes type safety
- Compiler cannot detect errors
- Runtime errors become more likely

Example:

```ts
let user: any = "Alex"

user() // Runtime error
```

---

# Best Practices

- Avoid using `any` unless necessary
- Replace `any` with specific types when possible
- Check types using `typeof`
- Document why `any` is used

---

# Better Alternative

Use `unknown` instead of `any` for safer handling.

```ts
let value: unknown = "hello"

if (typeof value === "string") {
    console.log(value.toUpperCase())
}
```

---

# Key Takeaways

- `any` allows storing any value
- It disables TypeScript type checking
- It should only be used temporarily
- Prefer specific types or `unknown` whenever possible
